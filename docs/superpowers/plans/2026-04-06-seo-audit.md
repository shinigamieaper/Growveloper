## Task 5: JSON-LD — resources detail page (`Article`)

The resources `[slug]` page is the only dynamic detail page without JSON-LD. `resource` (type `ResourcePageData`) has: `title`, `description`, `slug`, `publishedAt?`. These are all available after `if (!resource) notFound()`.

**Files:**
- Modify: `src/app/resources/[slug]/page.tsx`

- [ ] **Step 1: Add JSON-LD `Article` to `src/app/resources/[slug]/page.tsx`**

  In the `ResourcePage` default export, find the `return (` statement. It currently opens with:
  ```tsx
  return (
    <>
      {/* 01 — Header */}
      <section className="pt-32 pb-12 md:pt-40 md:pb-16">
  ```

  Insert the JSON-LD script block as the first child of the fragment:
  ```tsx
  return (
    <>
      <script
        type="application/ld+json"
        suppressHydrationWarning
        dangerouslySetInnerHTML={{
          __html: JSON.stringify({
            "@context": "https://schema.org",
            "@type": "Article",
            headline: resource.title,
            description: resource.description,
            ...(resource.publishedAt ? { datePublished: resource.publishedAt } : {}),
            url: `https://growveloper.com/resources/${resource.slug}`,
            publisher: {
              "@type": "Organization",
              name: "GROWVELOPER",
              url: "https://growveloper.com",
            },
          }),
        }}
      />
      {/* 01 — Header */}
      <section className="pt-32 pb-12 md:pt-40 md:pb-16">
  ```

  Note: `resource.publishedAt` is optional on `ResourcePageData` (it extends `ResourceCardData` which doesn't define it, but `ResourcePageData` adds it as `publishedAt?: string`). The conditional spread `...(resource.publishedAt ? { datePublished: resource.publishedAt } : {})` safely omits the field if undefined.
